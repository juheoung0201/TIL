# 브랜치를 나누는 방법
## 1. 브랜치 생성
~~~
$ git branch 브랜치명
~~~

## 2. 브랜치 변경
~~~
$ git checkout 브랜치명
~~~

## 3. 변경 브랜치에서 첫 push
~~~
$ git push -u origin 브랜치명
~~~

이후에는 그냥 push를 사용하면 된다.

### 왜 -u origin 브랜치명을 사용해야 할까?
branch를 내 로컬 환경에서만 만들었기 때문에 실제 깃에도 해당 branch와 연결해줄 필요가 있다. 따라서 경로를 설정하는 과정이라고 생각하면 될 것 같다.

## 4. 브랜치 삭제
우선 checkout을 통해 삭제 할 branch에서 빠져나온다. 
~~~
$ git branch -d 브랜치명
~~~
이때 해당 브랜치에서 push되지 않은 commit이 있는경우 오류가 발생한다.
이러한 상황에서 해당 브랜치를 삭제하고 싶은 경우에는
~~~
$ git branch -D 브랜치명
~~~
명령어를 통해 강제로 삭제할 수 있다.
